## 1. Що таке Freelance?

**Freelance** — це система заявок у платформі для передачі задач між відділами. Вона дозволяє Account Manager’ам делегувати роботу дизайнерам, SEO-спеціалістам, PPC-менеджерам і саппорту.

### **Для чого потрібна**:

- Оновлення дизайну лістингу (фото, банери).
- Запуск або зупинка PPC-реклами.
- Оптимізація SEO (ключові слова, заголовки).
- Вирішення питань із саппортом (відгуки, повернення).

> [!tip]  
> **Freelance** – система заявок для взаємодії з відділами дизайну, SEO, PPC, саппорту.

### **Як працює**:

- Менеджер створює заявку, вказуючи ASIN, тип задачі, дедлайн і бриф.
- Відповідальний виконавець (PPC, SEO тощо) отримує заявку, виконує її та надсилає результат.
- Менеджер перевіряє результат через кнопку “Результат” у карточці заявки.

> [!example]  
> Заявка на запуск PPC-реклами для “Golden Bracelet” із ключем “gold bracelet” і дедлайном 3 дні.

> [!info]  
> _Місце для скріншота_: Додайте зображення дашборду Freelance із вкладкою “Заявки в роботі”.  
> _Опис_: Покажіть, як виглядає список заявок.

## 2. Процес створення заявки

### **Кроки**:

1. У платформі відкрийте розділ **Freelance**.
2. Натисніть “Створити заявку” та виберіть тип послуги (PPC, SEO, дизайн, саппорт).
3. Вкажіть ASIN (наприклад, “Golden Bracelet”).
4. Заповніть бриф за шаблоном (отримайте у старшого менеджера).
    - Наприклад, для PPC: “Запустити рекламу за ключем ‘gold bracelet’, бюджет $10/день”.
5. Встановіть дедлайн (зазвичай 2–5 днів).
6. Надішліть заявку на перевірку старшому менеджеру.

### **Шаблони брифів**:

- Кожен тип заявки (PPC, SEO, дизайн) має шаблон із ключовими полями.
- Заповнюйте бриф чітко, щоб виконавець зрозумів задачу.

### **Дашборд заявок**:

- **Заявки в роботі**: Показує статус, дедлайни, непрочитані повідомлення.
- **Карточка заявки**: Містить чат із виконавцем, дедлайн, кнопку “Результат”.

> [!example]  
> У карточці заявки на PPC для “Golden Bracelet” PPC-менеджер написав: “Реклама запущена, ACOS 25%”.

> [!info]  
> _Місце для GIF_: Додайте GIF (3–5 секунд), що показує створення заявки у Freelance.  
> _Опис_: Демонстрація процесу вибору ASIN і заповнення брифу.

## 3. Практичне завдання

**Завдання**: Проаналізуйте шаблон брифу для PPC-заявки та підготуйте чернетку задачі для майбутньої роботи.  
**Інструкція**:

1. Отримайте шаблон брифу для PPC у старшого менеджера (уявіть, що він доступний).
2. Проаналізуйте поля шаблону:
    - Які дані потрібні (ASIN, ключові слова, бюджет)?
    - Які деталі важливі для PPC-менеджера?
3. Підготуйте чернетку брифу для “Golden Bracelet”:
    - Задача: запуск реклами за ключем “gold bracelet”.
    - Бюджет: $10/день.
    - Дедлайн: 3 дні.
4. Запишіть у нотатки:
    - Опис полів шаблону брифу.
    - Чернетка брифу для “Golden Bracelet”.
    - Висновок: як чіткий бриф допоможе PPC-менеджеру.

**Результат**:

- Зробіть скріншот: уявний шаблон брифу (або замалюйте структуру).
- Запишіть у нотатки:
    - Опис шаблону (поля, приклади).
    - Чернетка брифу.
    - Висновок про важливість брифу.

> [!info]  
> _Місце для скріншота_: Додайте структуру шаблону брифу або карточку заявки.  
> _Опис_: Покажіть, як виглядає бриф або заявка.

> Не створюйте заявки без апруву старшого менеджера. Усі брифи узгоджуйте перед відправкою.
