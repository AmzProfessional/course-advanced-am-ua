## 1. Dashboard: Центр моніторингу

**Dashboard** — це головний екран SellerBoard, який відображає ключові показники магазину чи окремого ASIN за вибраний період (день, тиждень, місяць). Він дозволяє швидко оцінити стан продажів і витрат.

![[Sellerboard-Review-2-1536x757.png]]

### **Ключові показники Dashboard**:

- **Замовлення**: Загальна кількість замовлень (органічні + рекламні).
- **Продажі**: Кількість проданих одиниць і дохід.
- **Повернення**: Кількість і вартість повернень.
- **Витрати на рекламу**: Витрати на PPC-кампанії (Sponsored Products).
- **Commissions**: FBA fees, referral fees, storage fees, inbound transportation.

### **Налаштування Dashboard**:

- **Період**: Вибір часового діапазону (вчора, тиждень, місяць, кастомний період).
- **Фільтри**: Аналіз за магазином, ASIN чи групою товарів.
- **Плитки**: Налаштування відображення (продажі, витрати, маржинальність).

### Дії:

1. У SellerBoard відкрийте **Dashboard**.
2. Виберіть період “Вчора” у верхньому меню.
3. Перегляньте плитки: замовлення, продажі, повернення, витрати на рекламу, commissions.
4. За потреби виберіть конкретний ASIN для детального аналізу.

**Приклад**: Для магазину “AquaFit Store” Dashboard за вчора показує: 50 проданих одиниць, $1000 доходу, 2 повернення ($20), $80 на рекламу, $150 commissions (FBA + referral fees). Це дозволяє швидко оцінити ефективність дня.

> **Візуалізація**:  
> _Місце для скріншота_: Додайте зображення Dashboard із плитками за вчорашній день.  
> _Опис_: Покажіть, як виглядає Dashboard із ключовими показниками.

## 2. Аналіз ключових показників

Dashboard спрощує щоденну перевірку, дозволяючи виявляти проблеми (високі витрати, повернення) і оцінювати ефективність продажів.

### **Що перевіряти**:

- **Замовлення та продажі**: Чи відповідає кількість проданих одиниць очікуванням (наприклад, 50+ одиниць для популярного ASIN).
- **Повернення**: Чи не перевищує норму (1–2% від продажів).
- **Витрати на рекламу**: Чи окупаються (порівняйте з доходом).
- **Commissions**: Чи немає аномальних списань (наприклад, високі storage fees).

### Дії:

1. На **Dashboard** виберіть період “Вчора” і магазин (наприклад, “AquaFit Store”).
2. Клацніть на плитку **Sales**, щоб переглянути розподіл органічних і рекламних продажів.
3. Перевірте **Advertising Costs** для оцінки витрат на PPC.
4. Відкрийте **Amazon Fees**, щоб проаналізувати FBA fees, referral fees, інші commissions.
5. Якщо виявлено проблему (наприклад, високі повернення), запишіть ASIN і повідомте старшого менеджера.

**Приклад**: Для ASIN “Bottle Holder” Dashboard показує 10 проданих одиниць, але 3 повернення ($18) — це 30% від продажів. Потрібно перевірити відгуки в Seller Central (**Customer Reviews**) і створити заявку (**Freelance** > **SEO**) для аналізу причин.

> **Візуалізація**:  
> _Місце для GIF_: Додайте GIF (3–5 секунд), що показує вибір періоду “Вчора” і огляд плиток Dashboard.  
> _Опис_: Демонстрація, як аналізувати показники на Dashboard.

## 3. Практичне завдання

**Завдання**: Налаштуйте Dashboard і проаналізуйте ключові показники за вчорашній день.  
**Інструкція**:

1. У SellerBoard відкрийте **Dashboard**, виберіть магазин (наприклад, “AquaFit Store”).
2. Налаштуйте період “Вчора” і перевірте:
    - Кількість замовлень і проданих одиниць.
    - Витрати на рекламу.
    - Кількість і вартість повернень.
    - Commissions (FBA fees, referral fees).
3. Виберіть один ASIN (наприклад, “Bottle Holder”) і проаналізуйте його показники.
4. Підготуйте звіт:
    - Опишіть показники за вчора (замовлення, продажі, витрати, повернення).
    - Вкажіть показники для вибраного ASIN.
    - Виявіть проблему (наприклад, високі повернення) і запропонуйте дії (заявка на SEO, перевірка в Seller Central).

**Результат**:

- Зробіть скріншоти: **Dashboard** за вчора, деталі для одного ASIN.
- Запишіть у нотатки в Obsidian:
    - Назва магазину.
    - Показники за вчора (наприклад, 50 одиниць, $80 реклама, 2 повернення).
    - Показники ASIN (наприклад, 10 одиниць, 3 повернення).
    - Виявлена проблема (наприклад, “30% повернень для Bottle Holder”).
    - Пропозиція для старшого менеджера (наприклад, “Перевірити відгуки, створити заявку на SEO”).

> **Візуалізація**:  
> _Місце для скріншота_: Додайте звіт із аналізом Dashboard і ASIN.  
> _Опис_: Покажіть, як виглядає структурований звіт.

## 5. Ключові поради для Account Manager’ів

- **Щоденність**: Перевіряйте Dashboard щоранку для оцінки ключових показників.
- **Періоди**: Аналізуйте “Вчора” для оперативної реакції, тиждень/місяць — для трендів.
- **Проблеми**: Звертайте увагу на високі повернення (>2%) чи витрати на рекламу без доходу.
- **Заявки**: Створюйте заявки (**Freelance** > **SEO**, **PPC**) для вирішення проблем.
- **Документація**: Зберігайте нотатки в Obsidian для відстеження змін.

> _Місце для GIF_: Додайте GIF, що показує аналіз продажів і витрат на Dashboard.
