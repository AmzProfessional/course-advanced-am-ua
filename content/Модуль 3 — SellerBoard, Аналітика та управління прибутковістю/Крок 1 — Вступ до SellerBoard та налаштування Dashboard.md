## 1. Значення утренньої перевірки

Утрення перевірка — це щоденний процес, який дозволяє Account Manager'ам оцінити стан магазину за попередній день. Основна мета — виявити аномалії (зростання чи падіння продажів, повернення, витрати на рекламу) і запропонувати рішення старшому менеджеру.

**Чому це важливо?**

- США завершує свої добові дані вранці за вашим часовим поясом, що дає повну картину за вчора.
- Виявлення аномалій дозволяє швидко реагувати, наприклад, коригувати ціни, рекламу чи запаси.
- Це основа для прийняття стратегічних рішень, які впливають на прибутковість.

## 2. Налаштування Dashboard для утренньої перевірки

Dashboard — ваш основний інструмент для швидкого аналізу. Правильне налаштування економить час і допомагає зосередитись на критичних метриках.

### Кроки налаштування:

1. **Вибір періоду**:
    - Для ранішньої перевірки оберіть "Вчора" для аналізу даних за попередній день.
    - Для порівняння трендів використовуйте "Останні 7 днів" або "30 днів".
2. **Вибір метрик для плиток**:
    - **Продані одиниці**: Кількість проданих товарів.
    - **Продажі**: Загальна виручка.
    - **Повернення**: Кількість і сума повернень.
    - **Витрати на рекламу (PPC)**: Сума витрат і ROI.
    - **Собівартість**: Витрати на товар і логістику.
    - **Маржа та прибуток**: Чистий прибуток і відсоток маржі.
    - **Утримання Amazon**: FBA fees, referral fees.
3. **Кастомізація таблиці**:
    - Увімкніть потрібні стовпці: продані одиниці, продажі, повернення, реклама, собівартість, маржа, ROI.
    - Вимкніть некритичні метрики (наприклад, сесії чи конверсія, які не оновлюються щодня).
4. **Збереження шаблону**: Збережіть налаштування для швидкого доступу.

> **Практична порада**: Налаштуйте Dashboard так, щоб ключові метрики (продажі, повернення, реклама) були зверху. Це дозволить за 5 хвилин оцінити стан магазину.

> **Візуалізація**:  
> _Місце для GIF_: Додайте GIF (5-7 секунд), який показує налаштування таблиці та вибір періоду "Вчора".  
> _Опис_: Демонстрація процесу перетягування плиток і вибору метрик.

## 3. Процес утренньої перевірки

Утрення перевірка складається з чотирьох етапів. Ви можете відстежувати прогрес за допомогою чекбоксів:

- [ ]  **Виявлення аномалій**: Знайти відхилення в продажах, поверненнях, рекламі чи маржі.
- [ ]  **Визначення причин**: Проаналізувати, чому виникла аномалія (зміни в листингу, конкуренти, реклама).
- [ ]  **Пошук рішень**: Запропонувати дії для виправлення чи посилення ситуації.
- [ ]  **Передача пропозицій**: Підготувати звіт для старшого менеджера.

### Кроки перевірки:

1. **Відкрийте SellerBoard і виберіть маркетплейс** (наприклад, США).
2. **Перевірте плитки Dashboard** за вчорашній день:
    - Порівняйте продажі, повернення та витрати на рекламу з попередніми днями.
    - Використовуйте вкладку **Trends** для аналізу трендів за 7-30 днів.
3. **Сортування по стовпцях**:
    - Сортуйте таблицю за проданими одиницями, поверненнями, рекламою, маржею чи ROI.
    - Шукайте аномалії (наприклад, різке падіння продажів чи сплеск повернень).
4. **Аналіз аномалій**:
    - Перейдіть до вкладки **Trends** або **Diagram** для детального аналізу конкретного ASIN.
    - Натисніть "More" у таблиці, щоб перевірити розподіл витрат (реклама, повернення, собівартість, утримання Amazon).
5. **Пропозиції**: Запропонуйте дії (наприклад, підняти ціну, оптимізувати рекламу, перевірити запаси).

> **Візуалізація**:  
> _Місце для скріншота_: Додайте приклад таблиці з відсортованими даними (наприклад, по проданим одиницям чи поверненням).  
> _Опис_: Покажіть, як виглядає аномалія (наприклад, 1 продаж замість середніх 8).

## 4. Практичне завдання

**Завдання**: Проведіть утренню перевірку для магазину в SellerBoard.  
**Інструкція**:

1. Увійдіть у SellerBoard і виберіть маркетплейс (наприклад, США).
2. Налаштуйте Dashboard для періоду "Вчора" із метриками: продані одиниці, продажі, повернення, реклама, маржа, собівартість.
3. Відкрийте таблицю та виконайте сортування за:
    - Проданими одиницями.
    - Поверненнями.
    - Витратами на рекламу.
    - Маржею та ROI.
4. Знайдіть щонайменше одну аномалію (наприклад, падіння продажів, високі повернення чи низький ROI).
5. Перейдіть до вкладки **Trends** або **Diagram**, щоб перевірити дані за 7-30 днів.
6. Визначте причину аномалії (наприклад, зміна ціни, конкурент запустив знижку, проблема з рекламою).
7. Запропонуйте рішення (наприклад, підняти ціну, оптимізувати рекламу, перевірити запаси).

**Результат**:

- Зробіть скріншот Dashboard і таблиці з аномалією.
- Запишіть у нотатки в Obsidian:
    - Опис аномалії (наприклад, "ASIN X продав 1 одиницю замість середніх 8").
    - Причина (наприклад, "Конкурент знизив ціну до $5").
    - Пропозиція (наприклад, "Запустити знижку або перевірити запаси конкурента").

> **Візуалізація**:  
> _Місце для скріншота_: Додайте приклад аномалії в таблиці (наприклад, низькі продажі чи високі повернення).  
> _Опис_: Покажіть, як виглядає тренд для ASIN із аномалією.

## 5. Ключові поради для Account Manager'ів

- **Швидкість і ефективність**: Утрення перевірка має займати 5-10 хвилин після ознайомлення з магазином. Сортування по стовпцях значно прискорює процес.
- **Фокус на аномаліях**: Звертайте увагу на різкі зміни (наприклад, продажі впали з 8 до 1 одиниці або повернення зросли до 3).
- **Перевірка причин**: Завжди аналізуйте, чи аномалія викликана вашим листингом (зміна ціни, реклама) чи діями конкурентів (знижки, out-of-stock).
- **Пропозиції**: Завжди надавайте конкретні рішення (наприклад, "Запустити рекламу з бюджетом $20/день" або "Знайти постачальника з нижчою собівартістю").
- **Повернення — вбивці прибутку**: Навіть 1 повернення може суттєво знизити маржу. Аналізуйте їх причини та пропонуйте дії (наприклад, покращення листингу).

> **Візуалізація**:  
> _Місце для GIF_: Додайте GIF, який показує сортування по стовпцях і перехід до **Trends** для аналізу аномалії.  
> _Опис_: Демонстрація швидкого пошуку аномалії (наприклад, падіння продажів).

## 6. Інтерактивний чекліст утренньої перевірки

Використовуйте цей чекліст у Obsidian для відстеження прогресу:

- [ ]  Виявлено аномалію (наприклад, низькі продажі, високі повернення, низький ROI).
- [ ]  Визначено причину (перевірено через **Trends**, **Diagram**, аналіз листингу чи конкурентів).
- [ ]  Запропоновано рішення (наприклад, оптимізація реклами, зміна ціни, перевірка запасів).
- [ ]  Підготовлено звіт для старшого менеджера з описом аномалії, причини та пропозицій.

> **Примітка**: Після затвердження пропозицій старшим менеджером виконайте дії (наприклад, запустіть рекламу чи замовте додаткові запаси).
